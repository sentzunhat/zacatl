# Patterns

Portable patterns for documentation discipline, coordination, and auditing across all projects.

## Files

| File                                                       | Topic                                                       |
| ---------------------------------------------------------- | ----------------------------------------------------------- |
| [evidence-discipline.md](evidence-discipline.md)           | Distinguish direct evidence from inference in documentation |
| [parallel-work-guardrails.md](parallel-work-guardrails.md) | Coordinate parallel work with reduced conflict              |
| [non-findings.md](non-findings.md)                         | Document negative results in bounded audits and reviews     |

## How to Use

- Use **evidence-discipline** when writing documentation, audits, or verification artifacts to maintain trust and clarity.
- Use **parallel-work-guardrails** when multiple humans or agents may edit the same files concurrently.
- Use **non-findings** when conducting bounded audits or reviews to show what was checked and what was not found.

## Portability

These patterns are not specific to HAWP. They apply to any project that values clear evidence, safe parallel coordination, and rigorous auditing.
