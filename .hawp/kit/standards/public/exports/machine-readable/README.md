# Machine-readable lineage (maintainers)

CSV and JSON files here record **standards promotion history** (classifications, path moves, absorb decisions).

## Downstream projects

**Do not use these files for day-to-day work.**

- Project and upstream labels are **redacted** (for example `upstream-a`, `upstream/archive/project-a/docs/`).
- Use canonical standards under `.hawp/kit/standards/` outside `public/`.
- Use `public/exports/hawp-absorbable/manifest.json` for approved new absorbs.

## Files

| File | Purpose |
| ---- | ------- |
| `standards-boundary-classification.csv` / `.json` | Path move and classification map for standards reorg |
| `legacy-standards-classification.csv` / `.json` | Older promotion candidate ledger (redacted labels) |
