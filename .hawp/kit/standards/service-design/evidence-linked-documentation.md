# Evidence-Linked Documentation

## Principle

Documentation should be linked to current implementation evidence and separated from future intent.

## Applies To

- standards updates
- architecture and boundary docs
- testing and operational expectations
- audit and promotion workflows

## Guidance

- Document what is implemented and verifiable today.
- Separate implemented behavior from planned improvements.
- Link standards to source evidence and audit records.

## Does Not Include

- project-specific implementation inventories
- unverified runtime assumptions
- product roadmap commitments
- internal-only security details
