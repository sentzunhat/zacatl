# HAWP Project Review Checklist

Use this checklist when reviewing HAWP docs, patterns, templates, and guidance for clarity and scope discipline.

## Core Shape

- [ ] Does the repo preserve Input -> Context -> Mission -> Constraints -> Output?
- [ ] Are new concepts optional unless explicitly accepted as core?
- [ ] Is checkpoint treated as an optional artifact, not a required field?

## Adoption

- [ ] Can a new user start in under 10 minutes?
- [ ] Is there a Micro HAWP template?
- [ ] Are examples easy to copy?

## Evidence

- [ ] Are direct evidence, inference, assumption, and unknown separated?
- [ ] Are unsupported claims avoided?
- [ ] Are non-findings included for audits where useful?

## Scope Control

- [ ] Does the guidance define what is out of scope?
- [ ] Does it prevent AI side quests?
- [ ] Does it avoid turning HAWP into a runtime framework?

## Maintenance

- [ ] Are duplicate docs removed or merged?
- [ ] Are templates clearly optional?
- [ ] Are README links current?
