# Public Safety Checklist

Use this checklist before publishing HAWP updates or sharing the repository broadly.

- [ ] No personal names are required to understand the docs.
- [ ] No private project or product names remain in public examples.
- [ ] No company, client, or customer-specific context is included unless intentionally public.
- [ ] No credentials, tokens, secrets, or local logs are present.
- [ ] No local file paths or environment details are included unless necessary and sanitized.
- [ ] No personal health, family, or financial context appears in public docs.
- [ ] Public examples are fictional or clearly generic.
- [ ] Templates remain reusable across projects.
- [ ] README text describes HAWP as a generic protocol, not a private workflow.
- [ ] Repo-local or private overlays are not presented as part of the public core.
- [ ] Status artifacts included in the repo were reviewed for privacy and portability.
- [ ] No private branch names, issue numbers, PR links, or commit SHAs are included unless intentionally public.
- [ ] No AI chat transcripts or generated logs are included unless manually reviewed and sanitized.
- [ ] Install and update snippets are either generic or clearly marked as public upstream distribution metadata.
