# Rotate npm publish token and enforce engine-strict installs

## Work Item

**UUID:** `754afa86-91f4-4a46-8094-756eab1f3f68`
**Type:** security
**Priority:** P1
**Reported:** 2026-07-13
**Source:** Security/arch/scalability audit — `.hawp/work/evidence/2026/07/13/AUDIT-002-security-arch-scalability-report.md`

## Investigation

Confirmed LOW (adversarially verified; requires maintainer action outside the
repo for step 1):

- A live npm publish token sits in plaintext in the maintainer's user-level
  `~/.npmrc` (npm's default login storage — conventional, but long-lived
  publish rights for @sentzunhat/zacatl), and the repo `.gitignore` does not
  ignore `.npmrc`, so a future project-level `.npmrc` could be committed.
- `engines: node >=26` is declared but unenforced (`package.json:12-15`) —
  consumers on older Node fail at runtime (`node:sqlite` import) instead of at
  install. Node 26 is also pre-LTS; the floor deserves an explicit decision.

## Plan

1. **Maintainer action:** rotate the npm token (granular automation token,
   shortest practical expiry); prefer CI-injected `NODE_AUTH_TOKEN` for
   publishes.
2. Add `.npmrc` to `.gitignore`.
3. Commit a project `.npmrc` containing only `engine-strict=true` (no auth) —
   note this conflicts with step 2's ignore rule; use `git add -f` once or
   ignore only `~/.npmrc`-style auth lines via a documented convention.
   Decide during review.
4. Record the Node 26 pre-LTS floor decision (keep — `node:sqlite` requires
   it) in the changelog/docs.
5. Supply-chain process note from the audit: several pinned majors exceed the
   publicly-known latest (js-yaml 5.x, uuid 14.x, …) — a dependency-confusion
   smell if ever built against a non-canonical registry (mitigated today by
   lockfile integrity hashes). Add a pre-publish registry cross-check and use
   `npm ci` in CI.

## Risk + Review Gate

**Risk:** low for the repo changes; token rotation must be coordinated so a
release is not attempted mid-rotation.
**Gate:** plan-ready; step 1 is maintainer-only.

## Verification

`npm install` on Node 24 fails fast with the engines error; publish dry-run
still passes on Node 26; no auth lines in any tracked file (`git grep _authToken`).
