## Investigation

Confirmed MED + LOWs (adversarially verified), batched as low-risk polish:

- node:sqlite prepared statements re-created on every query
  (`orm/nodesqlite/adapter.ts:113,137,176,211,240,258`) — SQL re-parsed and
  re-compiled per CRUD call despite the class docstring claiming prepared
  statements are cached "for performance" (MED).
- Express proxy `pathRewrite` regex built from the unescaped prefix
  (`express/api-adapter.ts:114-118`) — config metacharacters broaden matching
  (LOW, config-driven only).
- Fastify SPA fallback passes the raw undecoded URL to `sendFile`
  (`fastify/page-adapter.ts:22-32`) — query string becomes part of the
  filename (`app.js?v=1` 404s); explicit `..` rejection is delegated entirely
  to @fastify/static (LOW).
- Config/locale loaders are sync-only (`configuration/json.ts:9`,
  `yml.ts:11`, `localization/node/helpers.ts:51`) — fine at bootstrap, an
  event-loop stall if ever used per-request (LOW).

## Plan

1. Cache `StatementSync` handles per statement shape in the node:sqlite
   adapter (invalidate on table name change; table names are already
   regex-gated).
2. Regex-escape the prefix before building the Express proxy `pathRewrite`.
3. In the Fastify SPA fallback, parse the URL, use only the decoded pathname,
   drop the query string, and reject paths containing `..` explicitly.
4. Add `fs.promises` async variants of the config/locale loaders; document the
   sync variants as bootstrap-only.
