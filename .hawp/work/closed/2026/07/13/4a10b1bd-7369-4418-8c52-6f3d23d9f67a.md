# Sanitize Mongoose adapter filters and wrap updates in $set

**Status:** done and verified on 2026-07-13

**Type:** security
**Priority:** P1

**Source:** Security/arch/scalability audit — `.hawp/work/evidence/2026/07/13/AUDIT-002-security-arch-scalability-report.md`

## Investigation

Confirmed MED (adversarially verified).

- `src/service/layers/infrastructure/orm/mongoose/adapter.ts:91-95` —
  `findMany` forwards the caller's filter object verbatim to `model.find()`.
  If a consumer passes untrusted input, operator injection applies:
  `{ password: { $ne: null } }` auth-bypass shapes, `$where` JavaScript
  evaluation. The repository layer above is an equally thin passthrough.
- `adapter.ts:119-124` — `update` forwards operator keys, so an untrusted
  payload can `$unset`/`$rename` arbitrary fields.

## Plan

Take the easiest robust path — mongoose's built-in sanitization rather than a
new filter type:

1. Enable mongoose `sanitizeFilter: true` where the adapter issues queries
   (per-call option or connection-level `mongoose.set('sanitizeFilter', true)`),
   which neutralizes `$`-operator injection in filter values.
2. Additionally reject top-level `$` keys in adapter-received filters
   (`$where` arrives at the top level and is not covered by sanitizeFilter).
3. Wrap update payloads as `{ $set: update }` in the adapter unless the caller
   explicitly opts into raw operator updates via a typed escape hatch.
4. Document that repository filters are treated as data, not operators, and
   how to opt out deliberately.

## Risk + Review Gate

**Risk:** medium — consumers that intentionally passed operator filters/updates
through the repository layer will need the escape hatch; changelog callout.
**Gate:** plan-ready; explicit approval required before implementation.

## Verification

New adapter tests: `$ne`/`$gt` in filter values are treated as literals,
top-level `$where` rejected, plain update cannot `$unset`, escape hatch works.
Full test suite, type check, lint, mongodb example smoke test.
Verified locally with `npm test`, `npm run type:check`, and `npm run lint:silent`.
