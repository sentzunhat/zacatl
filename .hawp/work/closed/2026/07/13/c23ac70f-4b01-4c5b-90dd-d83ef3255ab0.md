# Clean up database driver dependency declarations

## Work Item

**UUID:** `c23ac70f-4b01-4c5b-90dd-d83ef3255ab0`
**Type:** improvement

**Reported:** 2026-07-13

## Investigation

Confirmed MED + LOWs (adversarially verified). The manifest declares a phantom
native-dependency surface:

- `better-sqlite3` appears in peerDependencies, optionalDependencies, types,
  and allowScripts (`package.json:177,183,189-191,1035`) with **zero imports**
  anywhere in src — consumers native-compile code that is never executed.
- `sqlite3` is declared and native-compiled solely to back the re-export barrel
  `src/third-party/databases/sqlite3.ts` (`package.json:180,186,1039`) —
  examples use it via `dialectModule`, but the framework itself never loads it.
- DB drivers sit in both optionalDependencies AND peerDependencies with
  mismatched ranges (`package.json:176-204`) — optional auto-install defeats
  the peer opt-in intent.
- `peerDependenciesMeta.sequelize` marks sequelize optional while sequelize is
  a hard dependency (`package.json:171,198-200`) — manifest contradicts itself.
- `i18n ^0.15.3` is a pre-1.0 hard runtime dependency (`package.json:166`).

## Plan

1. Remove all five better-sqlite3 references (peer, optional, types,
   allowScripts) unless a use case is identified — none found in the audit.
2. Decide sqlite3's role: keep as optional peer for the Sequelize sqlite
   dialect (examples need it) but drop it from optionalDependencies so it is
   not force-installed; keep or drop the re-export barrel accordingly.
3. Resolve the optional-vs-peer duplication: peers + `peerDependenciesMeta`
   optional flags only; align version ranges.
4. Fix the sequelize contradiction: either make sequelize a true optional peer
   (breaking, majorish) or delete the inert meta entry (non-breaking). Default:
   delete the meta entry now, revisit optionality separately.
5. Pin `i18n` exact.
6. Verify examples still install and run after the manifest changes.
