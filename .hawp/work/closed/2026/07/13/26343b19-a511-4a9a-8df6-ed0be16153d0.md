# Harden Docker images: digest pins, in-image healthcheck, SQLite volume path

## Work Item

**UUID:** `26343b19-a511-4a9a-8df6-ed0be16153d0`
**Type:** security
**Priority:** P1
**Reported:** 2026-07-13
**Source:** Security/arch/scalability audit — `.hawp/work/evidence/2026/07/13/AUDIT-002-security-arch-scalability-report.md`

## Investigation

The distroless migration itself was verified correct (commit `41d9a5f4` +
follow-ups). Remaining confirmed gaps:

- No digest pinning on base images (`examples/Dockerfile` FROM lines) and
  `mongo:latest` in the mongodb compose files — non-reproducible builds,
  silent major jumps (MED).
- Compose healthchecks call bare `node` with no fetch timeout and no HTTP
  status assertion — an HTTP 500 counts as healthy (LOW). No in-image
  `HEALTHCHECK` exists.
- SQLite examples write `database.sqlite` into the container-layer WORKDIR
  while the declared `./data:/app/data` volume goes unused
  (`examples/fastify-sqlite-react/docker-compose.yml:19-21`) — data lost on
  container recreation (LOW).
- Stale `.dockerignore` rules reference the old `backend/dist` layout
  (`.dockerignore:14-19`); broad `!build/**` widens the build context (LOW).

## Plan

1. Pin every `FROM` (`node:26-trixie-slim`, `gcr.io/distroless/nodejs26-debian13:nonroot`)
   and every compose image (`mongo`, `postgres`) by `@sha256:` digest; replace
   `mongo:latest` with a pinned major.
2. Add an in-image exec-form `HEALTHCHECK` using `/nodejs/bin/node -e` with
   `AbortSignal.timeout(4000)` and an `r.ok` exit assertion; align the compose
   healthchecks to the same command (reference design in the audit report §3).
3. Point SQLite examples at the volume: `DATABASE_URL=sqlite:/app/data/database.sqlite`,
   named volume writable by uid 65532; consider `read_only: true` + `tmpfs: /tmp`.
4. Prune `.dockerignore` to the actual builder inputs.
5. Rebuild all 8 example images and smoke-test each.

## Risk + Review Gate

**Risk:** low-medium — digest pins need a renovation habit (Renovate or manual);
DB path change affects local example data.
**Gate:** plan-ready.

## Verification

All 8 example images build; healthcheck flips unhealthy on a forced 500;
SQLite data survives `docker compose down && up`; `docker history` shows no
unpinned tags.
